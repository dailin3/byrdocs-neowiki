一、判断题（每题2分，共20分）

  1. 客观上讲，网络空间安全的根本原因是网络空间存在着漏洞或缺陷，且还存在网络攻击。
  2. 密码算法是公开的，保证密文信息不被破解关键就在于所使用密钥的安全。
  3. 公钥密码算法出现以后，对称密码算法在实际应用中已无使用价值了。
  4. 在 RSA 算法中，nnn 是公开的，且 n=p×qn=p\times qn=p×q（其中 ppp 和 qqq 都是素数)，所以，无论 nnn 为多大，ppp 或 qqq 都是很容易推算出来的。
  5. 哈希函数的安全性是指根据已知的哈希值不能推出对应的消息原文。
  6. 在数字签名方案中，消息签名值的长度与签名消息的长度无关。
  7. 恶意软件是指未经用户允许的情况下，在用户计算机中安装的软件，所以，用户计算机中通常不会有恶意软件的。
  8. 从防火墙发展来看，防火墙不仅要求具有很强的边界访问控制能力，而且还要求具有强大审计和自动分析能力。
  9. 目前，网络攻击的目的就是获取经济利益。

四、术语解释（共14分）

1. 认证性（4分）

› 答案 / 解析

确保一个消息的来源或消息本身被正确地标识，同时确保该标识没有被伪造，分为实体认证和消息认证。

2. 防火墙（4分）

› 答案 / 解析

一种高级访问控制设备，置于不同网络安全域之间，它通过相关的安全策略来控制（允许、拒绝、记录）进出网络的访问行为。

3. 数字签名（6分）

› 答案 / 解析

所谓数字签名（Digital Signature），也称电子签名，是指附加在数据单元上的一些数据，这些数据是利用签名算法首先对该数据单元进行关键信息提取，然后使用签名者的独有信息进行签名而形成的，用于标识签名者的身份以及签名者对该数据单元的认可，并能被接收者利用签名者的公有信息来验证该数据单元是否被篡改、伪造或抵赖。

五、简答题（共24分）

1. 公钥密码的基本思想及为什么能解决密钥分发

› 答案 / 解析
  1. 接受方拥有公私钥对，其中其公钥公开，发送方可获得接受方的公钥。
  2. 发送方采用公钥加密算法以接受方的公钥作为加密密钥对发送方发送的信息加密得密文。
  3. 发送方通过不安全信道将密文发送给接受方。
  4. 接受方收到密文后使用自己的私钥对密文解密还原出发送方发送的信息。

公钥密码体制能解决密钥分发问题：

  1. 传输的安全：由于使用接收方的公钥加密，攻击者即使得到密文也无法获得发送方发送的信息。
  2. 只有接收方才能得到发送方发送的信息。

2. Hash 函数的性质（包括特点和安全性）

› 答案 / 解析

输入任意长消息，输出固定长度的哈希值；

容易计算：对任何给定的消息，容易计算其哈希值。

单向性：对于给定的哈希值 hhh，要找到 MMM 使得 H(M)=hH(M)=hH(M)=h 在实际上是不可行的。

抗弱碰撞性：对于给定的消息 M1M_1M1​，要发现另一个消息 M2M_2M2​ 满足 H(M1)=H(M2)H(M_1)=H(M_2)H(M1​)=H(M2​) 在计算上是不可行的。

抗强碰撞性：找任意一对不同的消息 M1,M2M_1, M_2M1​,M2​ 使 H(M1)=H(M2)H(M_1)=H(M_2)H(M1​)=H(M2​) 在计算上是不可行的。

雪崩效应：当输入消息有一点变化时，输出的哈希值变化很大。

3. 计算机病毒的简评

› 答案 / 解析

网络化：计算机网络（互联网、物联网、通信网）成为计算机病毒的主要传播环境。

多样化：计算机病毒的传播途径多样化。

智能化：计算机病毒衍生速度快、寄生方式多、变化大。

黑客化：计算机病毒技术与黑客技术日益融合。

功利化：物质利益或特殊目的成为推动计算机病毒发展的最大动力。

4. 以 TCP 三次握手为例描述 DoS 攻击基本思想

› 答案 / 解析
  1. 客户端（攻击者）向它希望与之通信的服务器发送请求数据包，其中源地址是伪造的。
  2. 服务器能够接受请求，并按照伪造的源地址进行响应。
  3. 服务器等待客户端的回应，由于源地址是伪造的，服务器等不到回应，从而消耗资源。
