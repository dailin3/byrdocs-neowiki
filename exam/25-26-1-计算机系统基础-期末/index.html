1. 链接与库

  • main1.o 依赖 libx.a
  • main2.o 依赖 liby.a, libz.a
  • libz.a 依赖 liby.a
  1. gcc -static -o main1 main.o ./libx.a 命令会不会链接出错？如果会，分析原因并给出不会报错的正确写法。

8. 栈保护与缓冲区溢出

一个函数，它的 C 语言表述为：

int i;
int arr[5] = {0};
for (i = 0; i < 5; i++) {
    arr[i] = 0;
    printf("Hello World");
}
gets(arr);

同时提供了对应的汇编代码（未给出具体代码，仅描述）。

  1. 分析某奇怪的两句汇编语句的用途。分析另外两句奇怪的汇编语句的用途。它们组合起来有什么作用？（分别是用于设置金丝雀值/Canary 和检验金丝雀值是否被改变）
  2. 分析 i 存在栈的哪个位置？（以 rbp + 偏移量 的方式表示）
  3. 分析 arr[3] 的位置在哪里？（要求同上）
  4. 输入一段字母，请问输入多少会使函数出错？

9. UNIX I/O 与 进程控制

有两文件 t1.txt 和 t2.txt，分别有 “BUPTCT” 和 “CMUMIT” 内容。

函数设置一个 buf 字符型数组，容量为 6。创建三个文件描述符 fd1，fd2，fd3，分别指向 t1.txt, t2.txt, t2.txt。

过程如下：

  • 先读 fd1 中几个字符到 buf
  • 再读 fd3 几个字符到 buf
  • dup2(fd3, fd1)
  • 之后 fork
    • 若 fork 返回 0，则再从 fd2 读几个字符到 buf
    • 若 fork 返回非 0，则从 fd1 读几个字符到 buf
  • 分别在 dup2 前，fork == 0 分支，fork != 0 分支打印 buf 内容。

问题：

  1. 写出 buf 的三次内容。
  2. 每次进程启动时，都自动打开（ ）、（ ）、（ ）三个文件。
  3. 执行 fork 时，从（ ）态转到（ ）态（两空均填系统状态）。
